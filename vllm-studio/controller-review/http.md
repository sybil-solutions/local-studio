# Local Studio Controller — HTTP Layer (`src/http/`)

## 1. Purpose

This slice is the seam between the outside world and the Effect-based core of the controller. It builds the Hono application (CORS, logging, observability, rate limiting, API-key auth, OpenAPI docs), converts Effect programs into Hono handlers/middleware via a single `runControllerEffect` bridge, and provides small shared utilities used by route modules: bounded request-body readers, SSE stream helpers, and a typed loopback `fetch` wrapper for talking to local inference processes. Everything here is infrastructure; no business logic lives in this directory.

## 2. File-by-file walkthrough

### `src/http/app.ts` — application assembly (159 lines)

- **Exports:** `createApp(context, runtime)` (`app.ts:42`) and the `ControllerApplication` type (`app.ts:34-40`).
- **What it does:** Constructs one `Hono<ControllerEnvironment>` instance, installs global middleware in a fixed order, registers all route modules, then attaches OpenAPI docs and error handlers.
- **Middleware order matters** (`app.ts:49-88`):
  1. `controllerRuntimeMiddleware(runtime)` — stashes the Effect `ManagedRuntime` into Hono's context variables so every handler can run effects (`app.ts:49`).
  2. `cors(...)` — origin allow-list from `context.config.cors_origins`, defaulting to `[]` (deny all cross-origin). Exposes rate-limit headers (`app.ts:51-65`).
  3. A debug-log middleware written in Effect style: logs `METHOD path` unless the path is in `TELEMETRY_SKIP_PATHS`, then `Effect.andThen(Effect.tryPromise(next))` calls the rest of the chain (`app.ts:67-83`).
  4. Request observability (records every request to a SQLite store), mutating rate limit, read rate limit, mutating auth (`app.ts:85-88`).
- **Route registration:** `mergeRoutes(registerSystemRoutes(app, context), ...)` (`app.ts:90-103`). Each `register*Routes` returns the same `app` after chaining route definitions; `mergeRoutes` only performs a *type-level* merge (see `route-registrar.ts`). The inline `/health` route shows the canonical handler pattern: `app.get("/health", documentRoute, effectHandler((ctx) => Effect.succeed(ctx.json({ status: "ok" }))))` (`app.ts:98-102`).
- **Docs:** `/api/spec` serves an OpenAPI document generated by `hono-openapi`'s `openAPIRouteHandler` from the merged route app; `/api/docs` serves Swagger UI (`app.ts:105-128`). Note `exclude: ["/*", "/api/spec", "/api/docs"]` — the `"/*"` exclusion hides the proxy wildcard routes from the spec.
- **Error handling:** `notFound` returns `{ detail: "Not Found" }` 404 (`app.ts:130`). `onError` (`app.ts:132-156`) has three branches:
  - `isHttpStatus(error)` → typed HTTP error from `core/errors.ts`, serialized as `{ detail }` with its status.
  - Client-disconnect detection (`app.ts:138-153`): a long string-match list (`AbortError`, `ERR_STREAM_PREMATURE_CLOSE`, "stream was cancelled", …) maps mid-request client aborts to **status 499** (nginx's non-standard "client closed request") logged at debug level. This keeps SSE/proxy cancellations out of the error log.
  - Everything else → 500 "Internal Server Error" logged at error level. Note it does not log the stack, only `String(error)`.

### `src/http/route-registrar.ts` — type-level route merging (26 lines)

- **Exports:** `ControllerRouteApp` type, `documentRoute`, `defineRoutes`, `mergeRoutes`.
- `documentRoute` (`route-registrar.ts:8`) is a shared `hono-openapi` `describeRoute` middleware giving every route a generic "200 Successful response" in the spec. Route modules pass it as the second argument to `app.get/post/...`.
- `defineRoutes` (`route-registrar.ts:18-20`) is an identity function that constrains a registrar's return type so TypeScript keeps the precise chained-route type (needed for OpenAPI generation).
- `mergeRoutes` (`route-registrar.ts:22-26`) is the clever bit: at runtime it just returns `routes[0]` (all registrars mutate/chain the *same* Hono app), but its return type is `UnionToIntersection<Routes[number]>` — the classic contravariant-function trick (`route-registrar.ts:12-16`) that turns the union of all route-app types into an intersection, so `openAPIRouteHandler` sees every route's schema. **If you add a route module, you must add it both to the `mergeRoutes` call and to the `ControllerApplication` type in `app.ts:34-40`.**

### `src/http/effect-handler.ts` — the Effect↔Hono bridge (46 lines)

- **Exports:** `ControllerEffect<A, E>` (an `Effect` requiring `AppContextService`), `ControllerEnvironment` (Hono Variables shape), `controllerRuntimeMiddleware`, `effectHandler`, `effectMiddleware`.
- `ControllerEnvironment` (`effect-handler.ts:7-11`) declares that Hono's `ctx.get("controllerRuntime")` holds the `ControllerRuntime` — a `ManagedRuntime<AppContextService, AppContextInitializationError>` created once in `main.ts:60`.
- The heart is `runControllerEffect` (`effect-handler.ts:20-29`): runs the effect to an `Exit` via `runtime.runPromiseExit`, returns the value on success, otherwise rethrows the first defect/error as a plain JS exception (`Cause.findErrorOption` finds a typed error; `Cause.squash` collapses anything else). This is the boundary where Effect's typed-error world ends and Hono's exception world begins — thrown errors land in `app.onError`.
- `effectHandler` (`effect-handler.ts:31-36`) wraps a `(ctx) => Effect<Response>` into a Hono `Handler`. `effectMiddleware` (`effect-handler.ts:38-46`) does the same for `(ctx, next) => Effect<Response | void>` middleware. Because both return plain Promises, Hono needs no Effect knowledge.
- Consequence: route modules never call `Effect.runPromise` themselves; they return effects and let this bridge execute them with the shared runtime (so they get `AppContextService` provided and fiber supervision from the runtime).

### `src/http/security-middleware.ts` — auth + rate limiting (165 lines)

- **Exports:** `createMutatingAuthMiddleware`, `createMutatingRateLimitMiddleware`, `createReadRateLimitMiddleware`.
- **Constants** (`security-middleware.ts:7-20`): mutating = 120 req/60s per key; read = 1200 req/60s; read exemptions are `/health`, `/status`, `/metrics`, `/events`, `/api/docs`, `/api/spec`, plus any path ending in `/stream` or `/events` (`security-middleware.ts:27-31`) — SSE endpoints must not be throttled. Only `/health` (and OPTIONS preflights) skip auth entirely (`security-middleware.ts:8,35-36`).
- **Auth** (`security-middleware.ts:89-101`): if `config.api_key` is unset/empty, everything passes (the boot log in `main.ts:26` calls this "unauthenticated"). Otherwise it extracts `Authorization: Bearer <token>` or `X-API-Key` (`security-middleware.ts:64-72`) and compares with `timingSafeEqual` — with a length check first, since `timingSafeEqual` throws on mismatched lengths (`security-middleware.ts:74-81`). Failure → 401 with a `WWW-Authenticate` header. Note: despite the name, the middleware is mounted on `"*"` and guards **all** non-public methods/paths, not just mutating ones.
- **Rate limiting** (`security-middleware.ts:103-165`): in-memory fixed-window counters in two module-level `Map`s keyed by `${clientIp}:${METHOD}:${path}` (`security-middleware.ts:83-84`). Client IP comes from `cf-connecting-ip` → `x-real-ip` → **last** entry of `x-forwarded-for` (rightmost, i.e. the proxy-added address — `security-middleware.ts:38-49`), falling back to `"unknown"`. The mutating variant sets `X-RateLimit-*` headers on every mutating request; both variants return 429 + `Retry-After` when over the limit.
- **Memory safety:** `pruneRateLimitStore` (`security-middleware.ts:51-62`) caps each store at 10 000 entries: first deletes expired windows, then evicts oldest-insertion keys (Map iteration order). Because pruning happens only on allowed requests and keys are per-path, the map can hold up to 10k × small entries — fine for a local tool.
- `Effect.suspend` is used so the middleware body runs fresh per request rather than at middleware-construction time; `nextEffect` (`security-middleware.ts:86-87`) converts Hono's `next()` promise into an Effect.

### `src/http/observability-middleware.ts` — request telemetry (74 lines)

- **Exports:** `TELEMETRY_SKIP_PATHS`, `createControllerRequestObservabilityMiddleware`.
- **Skip list** (`observability-middleware.ts:7-14`): `/health`, `/metrics`, `/events`, `/status`, `/api/docs`, `/api/spec` — high-frequency/noise endpoints are neither logged nor recorded. The same set is reused by the debug-log middleware in `app.ts:71`.
- **Recording** (`observability-middleware.ts:31-73`): wraps `next()` and uses `Effect.onExit` so it records on success *and* failure. On success it derives the status from `ctx.res.status || 200` and marks success = 2xx/3xx. On failure it pulls the typed error out of the Effect `Cause` (`Cause.findErrorOption`, falling back to `Cause.squash`), mapping `HttpStatus` errors to their status and everything else to 500, and records `error_class` (e.g. `Http404` or the JS error name) plus `error_message` (`observability-middleware.ts:20-29`).
- Records go to `context.stores.controllerRequestStore.recordEffect(...)` (a SQLite-backed store; see `src/stores/controller-request-store.ts`), and the whole recording effect is `Effect.ignore`d — **telemetry can never fail a request**. Duration uses `performance.now()`.

### `src/http/bounded-body.ts` — DoS-safe body reading (109 lines)

- **Exports:** `RequestBodyTooLargeError`, `RequestBodyReadError` (both `Schema.TaggedErrorClass`), union type `RequestBodyError`, `readBoundedRequestBody`, `boundedFormData`.
- `readBoundedRequestBody(request, limit)` (`bounded-body.ts:46-87`):
  1. Fast path: reject from a declared `content-length` header over the limit (`bounded-body.ts:51-54`) — cheap, but headers can lie, so…
  2. Slow path: stream the body chunk-by-chunk with a `ReadableStreamDefaultReader`, accumulating `total`; the moment `total > limit` it cancels the reader and fails with `RequestBodyTooLargeError` (`bounded-body.ts:63-73`). No unbounded buffering ever happens.
  3. Resource safety: `Effect.acquireUseRelease` guarantees `reader.releaseLock()` runs on success, failure, or interruption (`bounded-body.ts:57-86`). Each chunk read is interruptible: `Effect.tryPromise` receives an AbortSignal and cancels the reader on Effect interruption (`bounded-body.ts:27-38`).
  4. Chunks are concatenated into a single `ArrayBuffer` at the end.
- `boundedFormData` (`bounded-body.ts:89-109`) reuses the bounded read, then reconstructs a synthetic `Request` from the buffered bytes and calls `.formData()` — a pragmatic trick to get the platform's multipart parser while keeping the size cap. Parse failures become `RequestBodyReadError`.
- Consumers: `modules/audio/routes.ts` (STT/TTS upload limits) and `modules/speech/routes.ts` (voice install/upload limits).

### `src/http/sse.ts` — Server-Sent Events helpers (42 lines)

- **Exports:** `toReadableByteStream`, `withSseHeartbeat`, `buildSseHeaders`.
- `toReadableByteStream` (`sse.ts:3-8`): maps an Effect `Stream<string>` through a `TextEncoder` into a web `ReadableStream<Uint8Array>` — the exact object a Hono/Bun `Response` body wants. Backpressure/cancellation of the web stream propagates into the Effect stream via `Stream.toReadableStream`.
- `withSseHeartbeat(frames, intervalMs, signal?)` (`sse.ts:21-34`): merges the real frame stream with `Stream.tick(intervalMs)` mapped to `": keepalive\n\n"` (an SSE comment line — ignored by clients but keeps proxies/NATs from killing the connection). `haltStrategy: "left"` means the merged stream lives only as long as the *frames* stream; heartbeats alone never keep it alive. If an `AbortSignal` is given, `Stream.interruptWhen(abortEffect(signal))` tears the whole stream down on client disconnect; `abortEffect` (`sse.ts:10-19`) is a textbook `Effect.callback` bridging a DOM event into an Effect, with a finalizer that removes the listener.
- `buildSseHeaders` (`sse.ts:36-42`): `text/event-stream`, `no-cache, no-transform`, `keep-alive`, and `X-Accel-Buffering: no` (disables nginx proxy buffering). Accepts extra headers (used e.g. to add `Last-Event-ID`-related fields in logs routes).
- Consumers: `modules/system/logs-routes.ts` (`/events`, log streams at lines 324/395) and other streaming routes.

### `src/http/local-fetch.ts` — typed loopback HTTP client (70 lines)

- **Exports:** `LocalFetchOptions` (`RequestInit` + `host?` + `timeoutMs?`), `LocalFetchError` (tagged error with `stage: "fetch" | "timeout"`), `fetchLocal`, `buildInferenceUrl`, `fetchInference`.
- `fetchLocal(port, path, options)` (`local-fetch.ts:21-57`): builds `http://{host}:{port}{path}` (default host `localhost`; `normalizePath` guarantees a leading `/`) and wraps `fetch` in `Effect.tryPromise`. The subtle part is signal composition (`local-fetch.ts:32`): if the caller passes their own `signal`, it is merged with Effect's interruption signal via `AbortSignal.any` — so either the caller aborting *or* the fiber being interrupted cancels the fetch.
- Timeouts (`local-fetch.ts:42-56`): `Effect.timeoutOrElse` converts a `timeoutMs` overrun into a `LocalFetchError` with `stage: "timeout"`, letting callers distinguish network failure from slowness without string matching.
- `buildInferenceUrl`/`fetchInference` (`local-fetch.ts:59-70`) are thin specializations reading `config.inference_host`/`config.inference_port` — the loopback address of the currently running inference engine. Consumers include `modules/proxy/*` (tokenization, OpenAI proxying) and `modules/system/engine-metrics-scrape.ts` (Prometheus `/metrics` scraping with a timeout).

## 3. How data/control flows

**Inbound request lifecycle** (the flow every request takes):

1. `Bun.serve({ fetch: app.fetch })` (`main.ts:50-55`) hands the raw request to the Hono app built by `createApp` (`app.ts:42`).
2. `controllerRuntimeMiddleware` stores the Effect runtime in context vars (`effect-handler.ts:13-18`).
3. CORS short-circuits disallowed origins / answers OPTIONS (`app.ts:51-65`).
4. Debug log (`app.ts:67-83`) → observability middleware starts its timer (`observability-middleware.ts:38`).
5. Rate limiters: mutating requests hit the 120/min bucket, non-exempt reads the 1200/min bucket (`security-middleware.ts:103-165`); 429 short-circuits here.
6. Auth: non-public requests need a matching bearer/API key *if* `api_key` is configured (`security-middleware.ts:89-101`); 401 short-circuits here.
7. A route handler — always an `effectHandler` — builds an `Effect` (typically `Effect.gen` reading JSON, calling stores/services, returning `ctx.json(...)`). `runControllerEffect` executes it on the shared `ManagedRuntime` (`effect-handler.ts:20-29`).
8. On the way out, the observability middleware's `Effect.onExit` records method/path/status/duration to SQLite, swallowing its own errors (`observability-middleware.ts:43-71`).
9. Failures: typed `HttpStatus` errors thrown from the exit become `{detail, status}` JSON (`app.ts:133-135`); client aborts become 499; anything else becomes 500 (`app.ts:132-156`).

**SSE flow:** a route handler builds an Effect `Stream<string>` of `data: ...\n\n` frames, wraps it in `withSseHeartbeat(frames, 15_000, signal)`, encodes via `toReadableByteStream`, and returns `new Response(stream, { headers: buildSseHeaders() })` (canonical example: `modules/system/logs-routes.ts:324-325`). Heartbeats keep idle connections alive; the client's disconnect signal interrupts the stream and the 499 branch in `onError` silences the resulting noise.

**Outbound-to-engine flow:** proxy/system modules call `fetchInference(context, "/v1/...", { signal: ctx.req.raw.signal, ... })` (`local-fetch.ts:62`), getting a `Response` they can stream straight back to the client or buffer.

**Body-upload flow:** routes that accept uploads call `readBoundedRequestBody`/`boundedFormData` with a module-specific byte cap; oversize bodies fail with `RequestBodyTooLargeError`, which the route maps to a 413-style `HttpStatus` error.

## 4. Key patterns & idioms

- **Effect boundary at the edge.** Inside, everything is `Effect<A, E, AppContextService>` with typed errors; at the Hono boundary `runControllerEffect` (`effect-handler.ts:20-29`) converts `Exit` back into value-or-throw. Learn `Exit.isSuccess`, `Cause.findErrorOption`, `Cause.squash` — they appear anywhere code must leave Effect-land.
- **`Schema.TaggedErrorClass`** (`core/errors.ts:3`, `bounded-body.ts:3`, `local-fetch.ts:6`): creates an error class with a `_tag` discriminant and schema-validated fields. Pattern-match via `instanceof` / `_tag`; `isHttpStatus` (`core/errors.ts:8`) is the canonical guard.
- **Middleware-as-Effect.** `effectMiddleware` lets middleware use `Effect.suspend`, `Effect.tryPromise`, `Effect.onExit` instead of async/await. `Effect.suspend` defers work to per-request time; `Effect.onExit` is "finally that also sees the failure"; `Effect.ignore` says "telemetry must not break the request".
- **Resource safety:** `Effect.acquireUseRelease` in `bounded-body.ts:57` and the `Effect.callback` + removal finalizer in `sse.ts:10-19` are the two idioms for "clean up even on fiber interruption".
- **Interruption = cancellation.** Effect's `tryPromise` hands you an AbortSignal; wiring it to `reader.cancel()` / `AbortSignal.any` (`bounded-body.ts:28-31`, `local-fetch.ts:32`) is how Ctrl-C / client disconnects propagate.
- **Hono route chaining as types.** `defineRoutes`/`mergeRoutes`/`UnionToIntersection` (`route-registrar.ts:12-26`) exist purely so `hono-openapi` can infer a full OpenAPI spec from chained route definitions; at runtime there is exactly one app object.
- **Module-level mutable state is acceptable here.** The rate-limit maps (`security-middleware.ts:24-25`) are plain module singletons — deliberate for a single-process local controller, but something to know before running multiple instances.
- **Fail-open vs fail-closed choices to notice:** auth fails *open* when no key is configured (local-first default); rate limiting fails *closed* (429); observability fails *silent* (`Effect.ignore`).

## 6. How to read this code

Suggested order:

1. **`route-registrar.ts`** — smallest file; understand `defineRoutes`/`mergeRoutes` first, because the "many registrars, one app" trick is confusing if you meet it in `app.ts` cold.
2. **`effect-handler.ts`** — the runtime bridge. `runControllerEffect` (lines 20-29) is the single most important function in the slice; everything else assumes you know where effects get executed and how typed errors become exceptions.
3. **`security-middleware.ts`** — self-contained, no Effect depth beyond `Effect.suspend`; good warm-up for the middleware style.
4. **`observability-middleware.ts`** — introduces `Effect.onExit` and `Cause` inspection in a small file.
5. **`app.ts`** — now read the assembly top-to-bottom: middleware order (49-88), route merge (90-103), docs (105-128), and the three-way `onError` (132-156). Cross-check each middleware against the files you just read.
6. **`sse.ts`** — the most Effect-idiomatic file (`Stream.tick`, `Stream.merge`, `Stream.interruptWhen`, `Effect.callback`); read with the Effect Stream docs nearby.
7. **`bounded-body.ts`** — `acquireUseRelease` + interruption-safe reader loop; the pattern to copy for any streaming resource.
8. **`local-fetch.ts`** — finish here; note `AbortSignal.any` and the `stage`-tagged timeout error.

First things to look for: (a) where the Effect world ends (the `runControllerEffect` bridge), (b) middleware *order* in `app.ts` — auth runs *after* rate limiting, and observability wraps both, (c) every short-circuit path (429/401/413/499) and which component owns it.

> Note on tests: there are no `*.test.ts` files under `src/http/` (verified); this slice is exercised indirectly through route-module tests elsewhere in the repo.
